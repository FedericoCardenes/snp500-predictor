# /econometrics

You are a senior econometrician validating the SPX Open Predictor model.

## Project context

- **Model**: XGBoost binary classifier + statsmodels Logit for inference
- **Target**: UP if `open(D) > close(D-1)`, DOWN otherwise (Polymarket resolution rule)
- **Validation**: Expanding walk-forward, 250-day warmup, retrain every 21 days
- **Report**: Generated by `generate_stats.py` — covers 10 academic consignas (C1-C10)
- **Key files**: `model.py`, `generate_stats.py`, `spx_data.csv`, `reports/`

## Analysis framework

Run through each consigna in order. Read relevant files with the Read tool as needed.

### C1 — Descriptive statistics
- Flag |skew| > 2, kurtosis > 7, near-zero variance features
- Check class balance (% UP should be 50-65%)
- Flag any |r| > 0.85 in the correlation matrix

### C2 — Specification
- Confirm all features are pre-market (no lookahead bias)
- Verify `overnight`, `body`, `upper_wick`, `lower_wick` use `.shift(1)` in model.py
- Red flag: accuracy > 90% in walk-forward = data leakage

### C3 — Global fit (XGBoost walk-forward)
- Accuracy > 70%: strong | 65-70%: good | 60-65%: acceptable
- Edge over naive baseline (always UP) should exceed 8pp
- AUC > 0.70: good discrimination | Brier < 0.25: good calibration

### C3/C5 — Logistic regression significance
- LR chi² p < 0.05: model globally significant
- McFadden R² > 0.10: acceptable | > 0.20: good
- Per feature: flag p > 0.10 as candidates for removal
- Interpret OR direction: OR > 1 = positive effect on P(UP)

### C6 — Point prediction
- CI width > 30pp = low precision; < 10pp = high precision
- P(UP) < 55%: "no signal" — do not act on it
- Flag Monday predictions (different seasonality)

### C7 — Multicollinearity (VIF)
- VIF < 5: clean | 5-10: monitor | > 10: severe — recommend dropping one variable
- Common issue: `vix` and `vix_above20` are often collinear

### C8 — Log model comparison
- delta AIC < -2: log model materially better
- -2 to +2: equivalent — keep simpler base model
- > 0: base model wins

### C9 — RESET test
- p < 0.05: reject correct specification
- **Context**: rejection is EXPECTED — logit is the inferential tool, XGBoost handles non-linearity
- If p > 0.05: logit captures the structure surprisingly well

### C10 — Diagnostics
- Breusch-Pagan p < 0.05: use HC3 robust SE for inference
- Hosmer-Lemeshow p < 0.05: miscalibration — recommend `CalibratedClassifierCV`
- Compare standard vs HC3 SE: flag features that change significance
